UX 101

Mar 27, 2019

The purpose of this document is to help a UX designer explain their role and process to a client (current or prospective) or a team. It’s intended for audiences that may still not understand the difference between UX design and UI design.

You’re welcome to use and change this document as you see fit, just don’t be an asshole about it :)

Download this page as an editable ODT doc

Definitions:

UX design (User Experience design) = Function (how the product works)

UI design (User Interface design) = Form (how the product looks)

UX Designer role
- Identifies users and usability needs, proposes practical design solutions for intuitive, pleasant product interaction (navigation, taxonomy, devices, task flows, feel)
- Creates the brief and requirements for the UI Designer to build

UI Designer role
- Creates the actual visual design and builds the User Interface

THE UX PROCESS

…is methodical, collaborative and iterative.

  1. Discover & Understand:

    Ask questions and analyse data until you understand the usability needs and the business goals.
    ⁃ What kinds of user types are there?
    ⁃ Who are the users? (demographics)
    ⁃ What are the problems we need to solve?
    ⁃ How can we add value?
    ⁃ What are the constraints?

  2. Strategy & Scope:

    Generate and document user needs and business goals, project requirements, resources and specifications.
    ⁃ What are we creating?
    ⁃ What can we realistically do?
    ⁃ When will we know we’re done with this iteration?
    ⁃ How will we measure success?

  3. Structure:

    Collaboratively create and validate information architecture, site map, navigation structure, taxonomies and task flows.
    ⁃ What are things called?
    ⁃ How do things need to be grouped?
    ⁃ What does each user type need to see on each screen?

  4. Wireframe:

    Iteratively create and validate the interface layout (keep the drafts content-based: no colours/fonts/images). If you have multiple user groups, create these with each group, then socialise with the other groups. Have the users present and explain to each other.

  5. Test & Refine:

    Test and socialise with the correct user groups and refine until functionality is validated.

  6. UI:

    Generate the brief for UI and visual design look and feel. Begin design iterations.
    ⁃ No design by committee. NO. DESIGN. BY. COMMITTEE.
    ⁃ Be prepared to explain and defend your designs, but be open to reasoned feedback. Remain convince-able.
    ⁃ Egos and opinions can be mitigated by A/B testing, user feedback and data.
    ⁃ Bad UI can ruin good UX

  7. Test & Refine:

    Test with the correct user groups and refine until ready to release.

  8. Monitor:

    Watch usage data and feedback channels

  9. Iterate:

    Commit to continuous improvement. Collect data and feedback, adjust and cycle again.
